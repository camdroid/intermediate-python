What is an API?

  • Application Programming Interface
  • Basically, it's a way to let other people do your work for you

Web Requests

  • HTTP is the "language of the web"
  • HTTP: HyperText Transfer Protocol
  • Basically, it's a format for how you send requests to website
  • We're working at a higher level than this, but we'll talk about types of requests

GET Requests

  • GET: "Get" information from another server
  • Read-only requests, don't change any data on the server
  • Something like fetching a web page, or pulling up your friends list on facebook

Parts of an API

  • API URL: First part - the web address where you can access the API
  • Endpoint: Route that tells the API which action you want to take
  • For example, we've seen get_recipes and add_recipe so far
  • Parameters: Same as we've seen before, only these need to be passed as a string now

Strings Everywhere

  • Everything on the web is sent as a string of characters
  • When we format parameters, we need to format them according to the API documentation
  • This page describes how to format arguments for the API we're using today

Writing a Command-Line Interface

  • Sometimes, you don't need a full web interface
  • Useful to be able to run scripts from your terminal
  • Let's go back to some of the boilerplate code we've been using...

Defining a Script

  • 
if __name__ == '__main__':
    app.run(debug=True)
  • This condition will be true when we run this script from the terminal
  • In this case, when we run this file from the terminal, we want the app to start running
  • Commonly, we'll also use 
    
if __name__ == '__main__':
    main()
  • Which means we need a function named main (which is a common pattern)

Getting arguments for our script

  • Python has a great module called argparse that makes it easier for us to get command-line arguments
  • Boilerplate
    
import argparse

parser = argparse.ArgumentParser()
# ...
args = parser.parse_args()
  • This creates an object to parse arguments that are passed in on the command line

Adding Arguments

  • There are two main types of arguments: postitional and optional (or named)
  • Similar to the types of python arguments we've seen earlier
  • Positional:
    
parser.addrgument('positional')
              
    
$ python cli_example.py value
  • Named:
    
parser.add_argument('--named')
              
    
$ python cli_example.py --named value

Argument Data Types

  • We can specify the data types of the arguments that we want to receive
  • 
parser.add_argument('--named', type=int)
  • Specify that the named argument is an integer, and we won't need to cast it when we read the arguments

Help!

  • Argparse makes a help section for you based on the arguments
  • Try running $ python cli_example.py --help
  • 
[cherringshaw@cherringshaw-24930 0318-sample (master) ]$ python cli_example.py --help
usage: cli_example.py [-h] [--optional OPTIONAL] [--number NUMBER] positional

positional arguments:
  positional           I'm a positional argument!

optional arguments:
  -h, --help           show this help message and exit
  --optional OPTIONAL  I'm an optional argument
  --number NUMBER      This is a number

Adding Help

  • In order to get these help messages, we use the "help" parameter in add_argument
  • 
parser.add_argument('--optional', help="I'm an optional argument")

General Tips for Building a Script

  • Use meaningful names for arguments
  • Break up your script into small functions that do one thing
  • Use tools like pdb to debug
