Flask Routes

  • A "route" is how you tell a server what program you want it to run.
  • @app.route('/') - means "run this code when someone requests '/' (the root directory)"
  • If we want to make other routes possible, we can add more lines like @app.route('my_route')
  • The route is what you enter in your web browser (after the server name)

    For example: https://twitter.com/camdroid217

    twitter.com specifies the server, and camdroid217 specifies that I want data for a particular user (me!)

Dynamic Routes

  • As you can imagine, if Twitter had to have a specific route for each user, that would be a huge amount of code!
  • To avoid this, we can use something called "dynamic routes."
  • We give the program an idea of what to expect, and it handles the rest!

Example


@app.route('/<username>')
def hello(username):
    return "Hello " + username

Templating

So far, not very useful - can only display basic sentences, nothing fancy.

Flask is a "back-end" language - it deals with what happens behind the scenes, not what the user sees.

Back End?

  • The web is split (mostly) into two types of development - "front end" and "back end"
  • Flask is a back-end language - it works on the servers, and isn't very useful for displaying results to the user (by itself)
  • To display things on a nice webpage, we turn to the front-end: HTML, CSS, and Javascript
  • In this class, we'll only focus on HTML, but CSS and JS are really useful!

  • Make all the things dynamic!
  • With render_template, we can use Python variables inside our templates!

Templating Example - Python


@app.route('/<username>')
def hello():
    return render_template('index.html', username=username)
